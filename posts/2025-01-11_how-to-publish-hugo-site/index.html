How to Publish a Hugo Site on GitHub Pages#

Publishing a Hugo site on GitHub Pages is a straightforward process. Follow these steps to get your site up and running.

Prerequisites#

  • Hugo installed on your machine
  • A GitHub account
  • Git installed on your machine

Step 1: Create a New Hugo Site#

First, create a new Hugo site by running the following command:

hugo new site my-hugo-site

Replace my-hugo-site with your desired site name.

Step 2: Add a Theme#

Navigate to your site directory and add a theme. For example, to add the Ananke theme:

cd my-hugo-site
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo 'theme = "ananke"' >> config.toml

Step 3: Create Content#

Create a new post to test your setup:

hugo new posts/my-first-post.md

Edit the generated file in content/posts/my-first-post.md and add some content.

Step 4: Build the Site#

Build your site with the following command:

hugo

The generated files will be in the public directory.

Step 5: Publish to GitHub Pages#

Option 1: User/Organization Pages#

  1. Create a repository named <username>.github.io on GitHub.
  2. Push the contents of the public directory to the repository:
cd public
git init
git remote add origin https://github.com/<username>/<username>.github.io.git
git add .
git commit -m "Initial commit"
git push -u origin master

Option 2: Project Pages#

  1. Create a repository for your project on GitHub.
  2. Add the repository as a submodule in the public directory:
cd my-hugo-site
git submodule add -b gh-pages https://github.com/<username>/<repository>.git public
  1. Generate the site and push to the gh-pages branch:
hugo
cd public
git add .
git commit -m "Publishing to gh-pages"
git push origin gh-pages

Conclusion#

Your Hugo site should now be live on GitHub Pages. Visit your GitHub Pages URL to see your site in action.

Happy publishing!
